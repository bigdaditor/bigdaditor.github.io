Context

Architecture

  • Client: React
  • Web Server: nginx
  • API Server: FastAPI (Python 3.12)
  • DB: MariaDB

요청 흐름

Client > Web Server > API Server > DB

Symptom

  • API Server URL로 요청을 직접 날렸을 땐 응답이 DevTools 에서 3초 내로 왔으나 같은 요청을 Web Server 통해서 요청 시 응답 속도가 7초 이상 걸리는 것을 확인
  • API Server와 DB Connection Pool 확인 했을 때 유지되고 있는 것을 확인
  • DevTools을 통해 Request Headers에서 Connection 값이 keep-alive 로 설정된 것을 확인

Cause

  • 구글링을 통해서 확인한 결과, Nginx 에서 proxy request header 설정 시 proxy_set_header Connction "" 을 명시하지 않으면 Connection: close 가 API Server로 요청이 전달 될 수 있음을 확인
  • 확인 하기위해 Nginx 코드 소스를 확인한 결과 proxy 요청 생성 시 Connection 헤더를 결정하는 로직에서 아래와 같이 close 값이 명시적으로 설정되는 부분을 확인했다.
# src/http/nginx_http_proxy_module.c
...

static ngx_int_t
ngx_http_proxy_internal_connection_variable(
    ngx_http_request_t *r,
    ngx_http_variable_value_t *v, 
    uintptr_t data
)
{

ngx_http_proxy_ctx_t *ctx;

ctx = ngx_http_get_module_ctx(r, ngx_http_proxy_module);

if (ctx == NULL || !ctx->legacy) {
    v->not_found = 1;
    return NGX_OK;
}

v->valid = 1;
v->no_cacheable = 0;
v->not_found = 0;
ngx_str_set(v, "close");

return NGX_OK;

}

Fix

  • Nginx proxy에서 API Server로 upstream 요청 시 Connection close 으로 설정하지 못하게 막기 위해 Connection "" 으로 설정

    HTTP/1.1 에서 Connection 헤더는 해당 요청과 응답 이후에 TCP 연결 동작 방식을 지시하는 역할을 한다. 기본값은 HTTP/1.1. 규칙에 따라 연결 유지(keep-alive)로 설정합니다. close는 TCP 응답 완료 후 연결 종료를 지시한다.

Result

  • 응답속도를 7초에서 3초 내외로 낮춤
