macOS 기준으로 Rust 설치부터 Tauri 세팅까지 한 번에 정리해보자. 블로그 글처럼 가볍게 읽으면서 따라 하면 된다.

1. Rust 설치하기 (rustup 사용)

Rust는 공식 버전 관리 도구인 rustup으로 설치하는 게 가장 편하다.

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

설치 후에는 터미널을 새로 열거나 아래를 실행해서 환경변수를 반영한다.

source "$HOME/.cargo/env"

정상 설치 여부는 이렇게 확인한다.

rustc --version
cargo --version

혹시 스크립트 내용을 확인하고 싶다면 아래 링크에서 직접 열어볼 수 있다.

https://sh.rustup.rs/

2. 기본 개발 도구 준비

Tauri는 Rust + WebView를 사용하는 데스크톱 앱 프레임워크라서 OS별 의존성이 있다. macOS에서는 Xcode Command Line Tools가 필요하다.

xcode-select --install

3. Tauri 프로젝트 세팅

Tauri는 create-tauri-app로 시작하는 게 제일 빠르다.

cargo install create-tauri-app
cargo create-tauri-app

질문이 나오면 아래처럼 선택하면 무난하다.

  • Project name: 원하는 이름
  • Package manager: 사용하는 것 선택 (npm, pnpm, yarn 등)
  • UI template: 기본 템플릿 (Vanilla/React/Vite 등)

프로젝트 디렉터리로 이동한 다음, 개발 서버를 띄운다.

cd <project-name>
cargo tauri dev

처음 실행 시 Rust 쪽 빌드가 일어나므로 조금 시간이 걸릴 수 있다.

4. Cargo.toml 주요 속성 한눈에 보기

Tauri 프로젝트의 Rust 설정은 src-tauri/Cargo.toml에 들어있다. 자주 보는 항목을 표로 정리하면 아래와 같다.

섹션/옵션 설명 예시
package 패키지 메타데이터 묶음 name, version, edition
package.name 크레이트 이름 my-tauri-app
package.version 버전 정보 (SemVer) 0.1.0
package.edition Rust 에디션 2021
dependencies 앱에서 사용하는 라이브러리 tauri, serde, tokio
build-dependencies 빌드 스크립트용 라이브러리 tauri-build
features 조건부 컴파일 플래그 custom-protocol, tray
profile 빌드 최적화 설정 profile.dev, profile.release
bin 실행 바이너리 설정 멀티 바이너리 구성

Cargo.toml은 “Rust 프로젝트 설정 파일”이라고 생각하면 쉽다. Tauri를 쓰더라도 결국 Rust 크레이트이기 때문에 이 파일이 핵심이다.
